Webhook – ruta

Pusiste path: "webhook/transaction" y path: "webhook/prospect".
En n8n, el nodo Webhook ya construye la URL como /webhook/<path>.
Arreglo: usa transaction y prospect (sin “webhook/”). 

Slack – falta resource y operation

Tus nodos Slack sólo tienen channel, text, username. El nodo necesita:

"resource": "message",
"operation": "post"


Arreglo: añade esos dos campos y deja channel (idealmente "#sales-notifications"), text, username. 

Email Send – parámetro inválido

Añadiste "mode": "sendEmail". El nodo emailSend no usa mode.
Arreglo: elimina mode. Deja toEmail, subject, text (y credenciales). 

Google Sheets – estructura de parámetros

Estás usando valueInputMode y un array data[{ range, values }]. Ese patrón no es el habitual del nodo (según versión). Lo estándar es operation: "append" (o “add” en v2) con sheetId, range y values (matriz).
Arreglo (simple y compatible):

"operation": "append",
"sheetId": "YOUR_SHEET_ID",
"range": "Sheet1!A:D",
"options": { "valueInputMode": "RAW" },
"values": "={{ [ [$json.date, $json.amount, $json.description, $json.transactionType] ] }}"


Y para “Prospects”: range: "Prospects!A:C" y values con [name, email, phone]. 

Resumen “semanal” mal planteado

El nodo Email - Weekly Summary está en la cadena de cada transacción, así que se enviará en cada POST, no semanal.
Arreglo: crea un nodo Cron semanal que dispare un sub-flujo que lea de Sheets (o calcule) y envíe el resumen. Quita ese email de la cadena del webhook. 

Secuenciación (mejor en paralelo)

Tras escribir en Sheets, lo lógico es notificar en paralelo a Slack y (si quieres) enviar un email. Ahora lo haces en cadena.
Arreglo: desde “Google Sheets - Add …” saca dos conexiones: a Slack y a Email (parallel fan-out). 

Mapping del payload del Webhook

Usas expresiones tipo {{$json["date"]}}. Eso funcionará si el cuerpo JSON llega en la raíz. Si tu frontend envía JSON como body, deberías mapear {{$json["body"]["date"]}}.
Arreglo: confirma el payload. Si dudas, usa un nodo Set de prueba o cambia a:

"={{$json.date || $json.body?.date}}"


(Si tu versión no soporta ?., usa un ternario: {{$json.date ? $json.date : $json["body"]["date"]}}). 

Credenciales

Normal que el export no las traiga. Pero recuerda asignar credenciales a cada nodo (Slack, Email/SMTP, Sheets) tras importar.

3) Resumen semanal (flujo aparte)

Añade un nodo Cron (FREQ=WEEKLY) → lee de Sheets (o de tu DB) → genera resumen → Email Send a team@asd.com.

 Slack con resource:"message" y operation:"post".

 Email sin mode.

 Google Sheets con operation:"append" y values.

 Revisa mapping del payload ($json.date vs $json.body.date).

 Mueve el resumen semanal a un flujo con Cron.
